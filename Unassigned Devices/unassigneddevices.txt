> **Unassigned Devices se denomina UD para abreviar.**
>
> Pase el ratón por encima de cualquier área activa de la página de la UD y un consejo sobre la herramienta le mostrará lo que hace al hacer clic en esa área.
> Puede montar dispositivos USB, unidades sata, recursos compartidos SMB/NFS remotos y archivos ISO con UD.  Cualquier dispositivo con el interruptor de montaje automático activado se montará cuando se inicie el array.  Todas las unidades y montajes SMB/NFS se desmontan cuando el array se detiene en el evento 'stopping_svcs'.
> Si quiere compartir su unidad, puede activar el interruptor de compartir.  El valor predeterminado para los recursos compartidos SMB remotos es el acceso público de lectura/escritura.  Active la seguridad SMB por usuario en la configuración de los dispositivos no asignados.  Los recursos compartidos NFS se exportan y el acceso es de lectura/escritura.  La exportación de dispositivos NFS se habilita en la configuración de Dispositivos no asignados. También se puede habilitar un script común que se ejecutará en cada montaje de disco.  Para compartir cualquier dispositivo UD, es necesario habilitar el uso compartido en la configuración de UD y activar el interruptor para compartir el dispositivo en particular.  Los recursos compartidos de eliminación SMB/NFS y los montajes ISO siempre se comparten.
> Después de introducir un punto de montaje, pulse &lt;Enter&gt; para guardar el cambio.
>
> Las opciones adicionales están disponibles cuando se hace clic en el botón <i style='color:black;font-weight:bold;' class="fa fa-plus-square"></i> icono por la identificación del dispositivo.  Cuando se desmonta el disco, se puede cambiar el punto de montaje del dispositivo.  La etiqueta del disco también se cambiará.  Se puede formatear un disco y ejecutar un fsck para comprobar el disco.
>
> La 'Configuración->Dispositivos no asignados' configuración de seguridad para SMB y NFS debe estar configurada correctamente para que el uso compartido de SMB y NFS funcione correctamente.
>
> Si el botón de montaje está en gris para un recurso compartido remoto SMB/NFS, el servidor remoto no responde a un ping.  El servidor remoto compartido debe responder a un ping o UD lo considerará fuera de línea.
>
> El botón "Añadir recurso compartido raíz" se desactiva cuando se activa "Habilitar recursos compartidos de disco" y cualquier recurso compartido raíz añadido previamente no se montará.
>
> La UD tiene un modo destructivo que permite borrar particiones de disco y formatear discos.	Si el modo destructivo no está activado en la configuración de la UD, NO podrá formatear un disco o eliminar particiones.  Vaya a 'Configuración->Dispositivos no asignados' para activar el modo destructivo.
>
> **Para formatear un disco:**
>
> + El modo destructivo debe estar activado.
> + El disco debe tener todas las particiones eliminadas.  Desmonte el disco, haga clic en el botón <i style='color:black;font-weight:bold;' class='fa fa-plus-square'></i> y haga clic en todos los <i style='color:red;font-weight:bold;' class='fa fa-remove hdd'></i> iconos para eliminar las particiones.
> + Si al disco se le ha hecho un preclear y muestra un botón gris de "Formato", haga clic en el botón <i style='color:black;font-weight:bold;' class='fa fa-plus-square'></i> y, a continuación, haga clic en el icono <i style='color:red;font-weight:bold;' class='fa fa-remove hdd'></i> para borrar el archivo de estado del preclear.
>
> **Nota: Un disco particionado en UD es compatible con el particionamiento del disco del Array y puede ser añadido a la misma.  Los formatos de archivo compatibles son XFS, XFS cifrado, BTRFS y BTRFS cifrado.**
>
> **Discos encriptados:**
>
> + Cualquier disco formateado en UD que esté encriptado utilizará la contraseña/frase de paso del array y podrá incorporarse al mismo.  O se puede establecer una contraseña única cuando se formatea.  Si establece una contraseña en el disco cuando se formatea, tendrá que introducir una contraseña de disco en Dispositivos no asignados->Configuración para ese disco.
> + Cualquier disco que no haya sido encriptado con la contraseña/frase de paso del Array puede montarse estableciendo una contraseña por disco en Dispositivos no asignados->Configuración.
>
> **Script de dispositivos no asignados:**
>
> UD incluye un script que se utiliza para montar y desmontar dispositivos y montajes SMB/NFS. Estos scripts se pueden utilizar desde la línea de comandos o en sus scripts según sea necesario.
>
> + '/usr/local/sbin/rc.unassigned mount autodevices' - todos los dispositivos configurados para el montaje automático se montarán.
> + '/usr/local/sbin/rc.unassigned mount autoshares' - se montarán todos los recuersos SMB/NFS configurados como montaje automático.
> + '/usr/local/sbin/rc.unassigned umount auto' - todos los dispositivos y recursos SMB/NFS configurados para el montaje automático serán desmontados.
> + '/usr/local/sbin/rc.unassigned umount all' - todos los dispositivos y recursos SMB/NFS se desmontan para preparar el apagado del Array.
> + '/usr/local/sbin/rc.unassigned mount /dev/sdX or devX' - montar el dispositivo sdX donde 'X' es el designador del dispositivo.  El devX es el nombre del dispositivo en la página UD.  Si el nombre del dispositivo es 'Dev 1', entonces use dev1 como el dispositivo a montar.
> + '/usr/local/sbin/rc.unassigned mount name=diskname' - montar el dispositivo por el nombre del disco.
> + '/usr/local/sbin/rc.unassigned umount /dev/sdX or devX' - desmontar dispositivo sdX donde 'X' es el designador del dispositivo. El devX es el nombre del dispositivo en la página UD.  Si el nombre del dispositivo es 'Dev 1', entonces use dev1 como el dispositivo a desmontar. Puede utilizar este comando en un script de UD para desmontar el dispositivo cuando el script haya finalizado.
> + '/usr/local/sbin/rc.unassigned umount name=diskname' - Desmontar dispositivo por nombre de disco.
> + '/usr/local/sbin/rc.unassigned spindown devX' - poner en reposo. El devX es el nombre del dispositivo en la página UD.  Si el nombre del dispositivo es 'Dev 1', entonces use dev1 como el dispositivo a poner en reposo.  Los discos SSD no se pondrán en reposo.
> + '/usr/local/sbin/rc.unassigned spindown name=diskname' - Poner en reposo un disco por nombre de disco.
> + '/usr/local/sbin/rc.unassigned mount SOURCE' - donde SOURCE es la fuente SMB/NFS.
> + '/usr/local/sbin/rc.unassigned umount SOURCE' - donde SOURCE es la fuente SMB/NFS.
>
> Tenga cuidado al usar la designación /dev/sdX porque puede cambiar después de un reinicio.
